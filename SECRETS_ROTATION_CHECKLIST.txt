CHECKLIST DE ROTACAO DE SEGREDOS
================================

Frequencia recomendada:
- Rotacao trimestral para chaves operacionais
- Rotacao imediata em caso de suspeita de exposicao

Matriz por ambiente:
- DEV: credenciais exclusivas de desenvolvimento
- HML: credenciais exclusivas de homologacao
- PROD: credenciais exclusivas de producao

Controles obrigatorios:
[ ] Nenhum segredo em codigo-fonte ou documentacao publica
[ ] Segredos armazenados em cofre/secret manager do provedor
[ ] Segredos com menor privilegio possivel
[ ] MFA habilitado para contas administrativas
[ ] Registro de data, responsavel e motivo da rotacao

Rotina operacional:
[ ] Executar: npm run compliance:secrets
[ ] Rotacionar chaves afetadas
[ ] Atualizar variaveis de ambiente por ambiente
[ ] Reiniciar servicos com janela controlada
[ ] Validar funcionamento pos-rotacao
[ ] Arquivar evidencias no dossie trimestral
